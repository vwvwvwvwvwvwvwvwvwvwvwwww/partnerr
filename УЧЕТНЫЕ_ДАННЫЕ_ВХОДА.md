Если вы создавали администратора вручную (`npm run create-admin`) или сотрудников (`npm run seed-staff`) **без** тех же переменных окружения, в базе могут быть **другие** пароли из вашего `backend/.env` или из командной строки.

## Если пароль «не подходит»

1. Проверьте, что открываете тот же URL, что в `APP_ORIGIN` / Vite (например `http://127.0.0.1:8848`), без смешения `localhost` и `127.0.0.1`.
2. Если не делали bootstrap, посмотрите **`ADMIN_PASSWORD`** в `backend/.env` и пересоздайте админа:  
   `npm run create-admin --prefix backend`  
   (переменные `ADMIN_USERNAME`, `ADMIN_PASSWORD`, `ADMIN_FULL_NAME` из окружения или `.env`).
3. Для сотрудников пароль задаётся только переменной **`SEED_STAFF_PASSWORD`** при запуске `seed-staff`:  
   `npm run seed-staff --prefix backend`.

## Безопасность (для диплома и продакшена)

Пароль `Admin12345Secure!` удобен для **локальной демонстрации**. Для реальной эксплуатации задайте свои значения в переменных окружения и заново выполните `create-admin` и `seed-staff`. Файл с реальными паролями не коммитьте в публичный репозиторий.
